HANAH Sua dúvida: ir além de document.body na raiz

Você perguntou se pode usar document.documentElement (que é o <html>), ou document.dataset, ou data-arch como root. Aqui está a análise:

document.documentElement — o <html>

const root = $('#root') || document.documentElement;


Sim, funciona. É o nó pai de tudo. Útil se você quiser que root.querySelectorAll(...) varra o documento inteiro incluindo <head>. Mas para TTS block scanning, é mais pesado que necessário.

document.body.dataset / data-arch

Você já usa isso via:

body.setAttribute('data-voice-arch', arch.id);


E o <body> tem class="field-closed mode-night". Isso é o state layer — não um root de query, mas sim um seletor CSS de estado global. Já está sendo usado corretamente.

document.querySelector('html') ou document.documentElement como root de scan

Para o scanBlocks(), o root serve para varrer o conteúdo legível. O <body> já é o correto. O <html> incluiria <head>, que não tem texto legível. Não há vantagem real.

O padrão que você pode expandir

Se quiser ir mais fundo no meta layer — tipo acessar data-arch de meta tags ou do <html> — você pode fazer:

// Lê data-arch do <html> element (raiz absoluta)
const archFromHTML = document.documentElement.dataset.arch;

// Lê meta theme-color ou meta custom
const metaArch = document.querySelector('meta[name="kob-arch"]')?.content;


E para escrever estado no root absoluto:

document.documentElement.setAttribute('data-arch', arch.id);
// ou
document.documentElement.dataset.arch = arch.id;


Isso é útil porque CSS consegue pegar com:

html[data-arch="atlas"] { --kob-voice-primary: #78e3ff; }
